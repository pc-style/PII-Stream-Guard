# Idea

PII Stream Guard exists to make screen sharing safe by default.

The app captures the screen in real time, detects likely PII on each frame, and masks it before the content leaves the machine. Instead of requiring manual review during calls, it adds a privacy layer directly into the capture pipeline.

Primary outcomes:

- protect emails, phone numbers, and user-defined needles by default;
- keep existing workflows with familiar tools (Zoom, Discord, OBS, etc.) by exposing a protected output stream/window;
- work in both local and remote modes so sensitive text can still be processed when the presenting and processing machines differ.

Key user value is reducing accidental leaks during demonstrations, demos, and support sessions, where information can appear on screen unexpectedly.
